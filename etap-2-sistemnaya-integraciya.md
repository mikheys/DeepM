# Этап 2 из 3: системная интеграция

Продолжение работы над локальным аналогом DeepL (этап 1 уже выполнен: есть рабочее ядро, двухпанельный переводчик и менеджер модели). Перед началом — изучи текущую структуру проекта (`/core`, `/os-integration`, `/ui`, `/config`, `/shortcuts`), чтобы новый код встраивался в уже заложенную архитектуру, а не дублировал её.

**Важное предупреждение по процессу:** код в этом этапе (низкоуровневые системные хуки) нельзя по-настоящему проверить без живого запуска на Windows с реальным вводом. Если ты работаешь в изолированной/облачной среде без доступа к реальной Windows-сессии — явно скажи об этом и предложи мне запустить и протестировать локально, прежде чем считать фичу готовой. Ожидай, что потребуется несколько итераций по фидбеку с реального использования — не пытайся довести до идеала за один проход.

## 1. Глобальный хоткей `Ctrl + тройное нажатие C`

- Реализуй низкоуровневый клавиатурный хук на Windows (`SetWindowsHookEx` через `windows-rs`, либо крейт `rdev`/`global-hotkey` — выбери и обоснуй), работающий в фоновом потоке, не блокирующем UI и не блокирующем обычную работу `Ctrl+C` в других приложениях.
- Логика распознавания: удержан `Ctrl` + три нажатия `C` в пределах разумного интервала (сделай интервал настраиваемым в конфиге, не хардкодь).
- При срабатывании: захватить буфер обмена (он уже содержит скопированный текст благодаря тройному `C`, который сам по себе работает как копирование), открыть/поднять главное окно, вставить туда этот текст как исходный для перевода. После операции **восстанови исходное содержимое буфера обмена**, если до этого там было что-то другое, что пользователь не ожидал бы потерять — обоснуй, нужно ли восстанавливать в данном случае (тройной Ctrl+C - это и есть копирование, так что скорее всего восстанавливать не нужно, но прокомментируй это решение).

## 2. Плавающая кнопка рядом с выделенным текстом

- При выделении текста пользователем в любом приложении — показать небольшую плавающую кнопку рядом с выделением (похожий паттерн на PopClip/Bob: либо через UI Automation API для точного определения границ выделения, либо через позицию курсора мыши в момент окончания drag-выделения как более простой fallback — выбери и обоснуй подход).
- По клику на кнопку — мгновенный перевод с показом результата во всплывающем тултипе рядом с кнопкой, без необходимости открывать главное окно.
- **Обязательно**: переключатель в настройках для полного отключения этой кнопки (раздел «Интерфейс» в настройках, добавленных на этапе 1/3).
- Учти конфликтные случаи: что происходит, если пользователь кликает мимо кнопки (она должна скрываться), что происходит при множественных быстрых выделениях подряд (не плодить несколько кнопок одновременно).

## 3. Хоткей «перевести выделенное и заменить на месте»

- Настраиваемый хоткей (дефолт предложи сам, но обязательно дай переопределить в настройках).
- При срабатывании: скопировать выделенный текст (эмуляция `Ctrl+C`), отправить в локальный движок перевода, эмулировать `Ctrl+V` для вставки перевода на место исходного текста, восстановить исходное содержимое буфера обмена после операции (здесь восстановление уже точно нужно, так как пользователь не просил копировать — он просил заменить текст на месте).
- Добавь короткую визуальную/звуковую обратную связь (по желанию пользователя в настройках) о том, что перевод происходит — операция не мгновенная, нужно дать понять, что она в процессе.

## 4. Автозапуск и трей

- Настройка автозапуска приложения с ОС (через `tauri-plugin-autostart` или аналог), переключатель в настройках.
- Иконка в системном трее с меню: открыть главное окно, включить/выключить плавающую кнопку, статус локальной модели (загружена / качается / ошибка), выход.
- Опция «запускать в трей, не открывая окно» при автозапуске.

## 5. Слой конфигурации хоткеев

- Все комбинации клавиш из пп. 1–3 должны быть **полностью переопределяемы пользователем** через экран настроек «Горячие клавиши» — никаких хардкод-комбинаций в коде. Это используй заложенный на этапе 1 модуль `/shortcuts`, доведи его до полноценной реализации.
- Обработай конфликты хоткеев (если пользователь пытается назначить уже занятую комбинацию — предупреди, не давай тихо перезаписать).

## 6. Критерий готовности этапа

- Приложение, запущенное в фоне/трее, реагирует на тройной `Ctrl+C` из любого приложения, открывая окно с уже вставленным выделенным текстом.
- При выделении текста где угодно появляется кнопка мгновенного перевода (и полностью пропадает, если выключена в настройках).
- Хоткей замены работает на месте в произвольном приложении (протестировать минимум в браузере и в текстовом редакторе/блокноте).
- Автозапуск и трей работают.

## 7. Если упёрся в системное ограничение

Если для какой-то из фич понадобятся специальные права (запуск от администратора, подпись драйвера, разрешения Accessibility и т.п.), или конкретный API ведёт себя нестабильно в разных версиях Windows — остановись и опиши проблему мне, не упрощай задачу тихо (например, не убирай восстановление буфера обмена «для простоты» без предупреждения).
